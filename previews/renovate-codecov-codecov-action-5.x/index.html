researcharr documentation

Welcome — this repository contains the documentation that powers the project website and GitHub Pages site. Use these pages for quick onboarding, deployment guidance, CI notes, and operational reference.

Below is a suggested table of contents (order follows the README and operator/developer flow):

Table of contents

  1. Getting Started — Quick start for running researcharr locally or in Docker, initial credentials, and basic configuration.
  2. Environment variables — All runtime environment variables (PUID, PGID, TIMEZONE, WEBUI_PORT, and more) with examples.
  3. Deployment and Resource Recommendations — Docker / Compose / Kubernetes examples and job/runtime caps (JOB_TIMEOUT, JOB_RLIMIT_AS_MB, JOB_RLIMIT_CPU_SECONDS).
  4. Health and Metrics — /health and /metrics endpoints, Docker healthcheck examples, and metrics exposed by the app.
  5. Versioning & Releases — Image labels, /api/version, CI tagging and best practices for tracing builds.
  6. Packaging and Distribution — Packaging targets, CI packaging jobs, and recommended approaches for releases.
  7. CI and Development — CI checks, formatting, pre-commit guidance, and branch/PR workflow for contributors.
  8. Contributing — Contributor guide, branching model, tests, and pre-commit checklist.
  9. PR Instructions — Quick commands and PR template guidance for submitting changes.
  10. Changelog — Canonical changelog and recent release notes.
  11. Recent Changes — Short summary of recent repo changes contributors should know about.
  12. Roadmap — Project priorities and near-term goals.
  13. FAQ — Frequently asked questions and troubleshooting tips.

How this page is used

  • GitHub Pages: The docs/ folder is published to GitHub Pages. index.md is the landing page used when the docs site is served from this folder.
